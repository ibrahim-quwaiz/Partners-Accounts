أنت مصمم واجهات وتجربة مستخدم (UI/UX) ومطور واجهة أمامية فقط.
لدي تطبيق محاسبي بسيط لشريكين (P1 و P2).
الواجهة السابقة كانت Streamlit، والآن أريد إعادة تصميم الواجهة فقط بواجهة ويب حديثة.
لا أريد تعديل الباكند أو منطق قاعدة البيانات، فقط الديزاين والواجهة.

معلومة مهمة:
- المستخدم لديه مشروعين داخل نفس التطبيق.
- نفس الإجراءات ونفس الجداول تُستخدم لكلا المشروعين.
- يفضل أن تكون قاعدة البيانات واحدة مع عزل البيانات عبر project_id.
- الواجهة يجب أن تدعم اختيار المشروع وتغيير السياق تلقائيًا.

هدف التصميم:
- واجهة عربية بالكامل (RTL).
- بسيطة، واضحة، مناسبة لتطبيق محاسبي داخلي.
- بدون تعقيد أو تحسينات مبكرة.

الجداول الموجودة:
- expenses (مصروفات)
- revenues (إيرادات)
- settlements (تسويات)
- periods (فترات)
- partners (شركاء)

المطلوب تصميمه:

1) التخطيط العام (Layout):
   - شريط علوي يحتوي على:
     - اسم التطبيق
     - اختيار المشروع (قائمة منسدلة)
     - اختيار الفترة
   - تنقل واضح بين الصفحات:
     - المعاملات
     - التقارير
     - الإشعارات (قريبًا)

2) صفحة المعاملات (الأهم):
   - تبويبات:
     - المصروفات
     - الإيرادات
     - التسويات

   داخل كل تبويب:
   - زر واضح "إضافة"
   - نموذج إضافة مرتب (Modal)
   - جدول يعرض البيانات (تاريخ – وصف – مبلغ – الشريك)
   - أزرار لكل سجل:
     - تعديل
     - حذف

   ⚠️ مهم جدًا:
   - الحذف لا يتم مباشرة.
   - عند الضغط على "حذف" يظهر مربع حوار تأكيد:
     - رسالة واضحة: "هل أنت متأكد من حذف هذا السجل؟"
     - زرين: "تأكيد الحذف" / "إلغاء"

3) صفحة التقارير:
   - تصميم مبدئي فقط (Placeholder)
   - كروت ملخص:
     - إجمالي المصروفات
     - إجمالي الإيرادات
     - الصافي
   - فلاتر: المشروع + الفترة

4) صفحة الإشعارات:
   - تصميم مبدئي فقط
   - نص: "سيتم إضافة الإشعارات لاحقًا"

5) تجربة المستخدم:
   - رسائل نجاح وخطأ واضحة بالعربية
   - تصميم نظيف Minimal
   - استخدام Cards / Dividers
   - دعم RTL كامل (الاتجاه، المحاذاة، الأيقونات)

6) المخرجات المطلوبة:
   - اقتراح تقنية واجهة مناسبة (React / Next.js)
   - هيكل ملفات Frontend منظم
   - مكونات قابلة لإعادة الاستخدام:
     - اختيار المشروع
     - اختيار الفترة
     - جدول البيانات
     - مربع تأكيد الحذف
   - ألوان هادئة مناسبة لتطبيق محاسبي (بدون بهرجة)

قيود مهمة:
- لا تضف تسجيل دخول الآن
- لا تضف صلاحيات
- لا تضف إشعارات فعلية
- لا تغيّر منطق الباكند
- ركّز على وضوح الواجهة وسهولة الاستخدام

ابدأ بتصميم صفحة "المعاملات" كاملة أولًا، ثم اعرض تصورًا لباقي الصفحات.
