أنت تعمل في Replit AI بوضع Design.
مهمتك تصميم وتحسين واجهات المستخدم (UI/UX) لمشروع React موجود بالفعل.

نحن الآن في مرحلة تصميم الصفحات وشكل المشروع فقط،
مع السماح بمنطق خفيف للعرض (Mock / UI State) بدون فرض منطق نهائي.

────────────────────────────
القواعد العامة
────────────────────────────
- مسموح استخدام state بسيط (tabs, dropdowns, modals).
- مسموح استخدام بيانات وهمية (Mock data) للعرض فقط.
- مسموح حسابات شكلية للعرض (UI only).
- لا تفترض أي منطق محاسبي نهائي.
- لا تعتمد على API أو Backend حقيقي.
- لا تغيّر بنية المشروع أو أسماء المسارات بدون سبب.

قبل أي تعديل:
- تحقّق هل الصفحة أو الملف موجود مسبقًا.
- إذا كان موجودًا:
  ✔ عدّل التصميم فقط (Layout / Colors / Spacing / Components).
  ✖ لا تكسر الوظيفة الحالية.
- إذا غير موجود:
  ✔ أنشئه بتصميم متناسق مع بقية المشروع.

────────────────────────────
فكرة التطبيق (للتصميم فقط)
────────────────────────────
برنامج محاسبي لإدارة شراكة بين شخصين على مشروعين فقط.

────────────────────────────
تدفق التطبيق (UI Flow)
────────────────────────────
1) صفحة تسجيل الدخول
2) صفحة اختيار المشروع
3) التطبيق الرئيسي (Topbar + Sidebar + محتوى)

────────────────────────────
الصفحات المطلوبة
(تحقّق من وجودها قبل الإنشاء)
────────────────────────────

0) صفحة تسجيل الدخول
- حقول:
  - البريد الإلكتروني / اسم المستخدم
  - كلمة المرور
- زر "دخول"
- تصميم عصري وبسيط
- لا تحقق حقيقي (UI فقط)

1) صفحة اختيار المشروع
- بطاقتان (Cards) فقط:
  - مشروع 1
  - مشروع 2
- أيقونة + اسم المشروع
- زر "الدخول للمشروع"
- الهدف: منع الخلط بين المشاريع

2) صفحة المعاملات
- Tabs:
  - إيرادات
  - مصروفات
  - تسويات
- Forms شكلية فقط
- استخدام state للتبديل بين التابات
- لا حسابات حقيقية

3) صفحة الفترات
- جدول فترات
- أعمدة مثل:
  - اسم الفترة
  - الحالة (مفتوحة / مغلقة)
- أزرار شكلية:
  - فتح فترة
  - إغلاق فترة

4) صفحة التقارير
- اختيار الفترة:
  - فترة محددة
  - أو "كل الفترات"
- Summary Cards (قيم وهمية):
  - إجمالي الإيرادات
  - إجمالي المصروفات
  - صافي الربح
- Tabs:
  - تقرير المصروفات
  - تقرير الإيرادات
  - تقرير التسويات
- جداول قراءة فقط

5) صفحة الإشعارات
- جدول إشعارات
- حالات شكلية:
  - Pending
  - Sent
  - Failed
- زر "إرسال" (UI فقط)

6) صفحة المستخدمين
- مستخدمين فقط
- اعرض:
  - الاسم
  - البريد الإلكتروني
  - رقم الجوال
  - الدور (ADMIN / TX_ONLY)
- لا تعرض كلمات المرور
- أزرار تعديل شكلية فقط

────────────────────────────
متطلبات التصميم
────────────────────────────
- اللغة عربية (RTL).
- متجاوب (Desktop + Mobile).
- تصميم عصري وبسيط.
- ألوان هادئة.
- وضوح الاستخدام أهم من الزخرفة.

────────────────────────────
بعد الانتهاء
────────────────────────────
- اقترح تحسينات UI/UX إن وُجدت (اختياري).
- لا تطبّق التحسينات تلقائيًا.
- أشر إلى نقاط الربط المستقبلية بـ TODO comments.

نحن حاليًا نركّز على:
الشكل، تجربة المستخدم، وترتيب الصفحات فقط.
